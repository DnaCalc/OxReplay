# Handoff Packet Template

Use this structure for new handoff packets.

## Header
1. handoff id
2. date
3. from repo
4. to repo
5. related workset or feature id

## Purpose
State the exact contract, doctrine, or design issue being handed off.

## Proposed change
State the normative or implementation-facing change clearly and concretely.

## Current evidence
List the replay artifacts, conformance outputs, or concrete examples that motivate the handoff.

## Impact
1. capability impact,
2. pack impact,
3. migration or fallback impact,
4. affected repos or hosts.

## Requested response
State the exact acknowledgement, decision, or upstream edit needed.
